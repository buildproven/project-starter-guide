# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in this project, please report it responsibly by emailing **<security@buildproven.ai>** instead of using the public issue tracker.

### What to Include in Your Report

Please provide:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fixes or patches

## Security Standards

This project implements security best practices including:

- **Dependency Scanning** - Regular updates and vulnerability monitoring
- **Code Review** - All contributions are reviewed for security issues
- **ESLint Security Rules** - Automated linting to catch common security problems
- **No Hardcoded Secrets** - Environment variables for sensitive configuration
- **HTTPS Only** - All resources use secure connections where applicable

## Supported Versions

Security updates are provided for:

- The latest major version
- The current stable release

## Security Advisories

Security advisories will be published on the [GitHub Security Advisories](https://github.com/buildproven/project-starter-guide/security/advisories) page.
